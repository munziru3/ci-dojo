Apa sih itu Codeigniter?

ditulis oleh Ezra Lazuardy pada 20 Mei 2019

Jadi gini, si api merah alias CodeIgniter Rocks (disingkat CI) itu sebuah framework PHP, untuk mempermudah pembuatan aplikasi web, dengan bahasa PHP tentunya. Eh tapi tunggu dulu.., arti dari kata "Framework" itu apaan sih?.

Framework atau dalam bahasa indonesia dapat diartikan sebagai “kerangka kerja” merupakan kumpulan dari fungsi-fungsi/prosedur-prosedur dan class-class untuk tujuan tertentu yang sudah siap digunakan sehingga bisa lebih mempermudah dan mempercepat pekerjaan seorang programer, tanpa harus membuat fungsi atau class dari awal.

Intinya, framework itu kumpulan kode-kode yang dibuat sedemikian rupa untuk mempermudah kita dalam membuat sebuah aplikasi. Dalam bidang pemrograman, banyak kok macam-macam Framework, dengan bahasa pemrograman yang beda pula. Contohnya Flutter dengan bahasa pemrograman Dart buat bikin aplikasi mobile native, Angular buat aplikasi web sekaligus mobile dengan bahasa TypeScript, dan si Laravel untuk buat aplikasi web dengan bahasa PHP. Berarti Laravel sama kayak CI dong?. Yup, mereka sama sama framework untuk membuat aplikasi web dengan bahasa PHP.

"Rival" nya CodeIgniter

Dalam pemrograman, penggunaan framework ini ada kelebihan dan kekurangannya juga. Tak hanya itu, seringkali diperdebatkan oleh para developer tentang framework apa yang menurutnya terbaik untuk membuat aplikasi. Contohnya ini nih, perdebatan antara CodeIgniter dan rivalnya, si Laravel. Hmmm, emang Laravel bagus dimananya sih?
